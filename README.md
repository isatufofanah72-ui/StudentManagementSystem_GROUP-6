# Structured Digital Solution for Public Service: Student Academic Performance Portal An enterprise-grade, modular desktop application designed to streamline student performance data management, ensure academic grading integrity, and provide real-time visual metrics. This system is tailored for higher education administration within Sierra Leone and directly addresses **UN Sustainable Development Goal 4 (SDG 4): Quality Education**. --- ## 🎯 Project Overview & Rationale Educational institutions face massive inefficiencies when relying on manual ledger indexing or scattered offline spreadsheets. These outdated frameworks introduce transcribing vulnerability, lag in reporting pipelines, and zero protective input bounds. This application builds a secure digital infrastructure providing: * **Automated Computation:** Instantly merges continuous assessments and exams into bounded final weights and strict GPA assignments. * **Demographic Analytics:** Provides instant administrator visibility into historical status distribution, gender percentage splits, and growth timelines. * **Interoperability & Scalability:** Powered by a relational **PostgreSQL** backend, enabling cross-system connectivity and secure concurrent operations. * **Public Service Reporting:** Compiles filtered tracking matrices instantly into formal, standardized PDF sheets for institutional archiving. ### 🌍 SDG 4 Alignment By bounding score inputs to explicit ranges (Assignments 0–40, Exams 0–60), validating entries through protective decision trees, and removing calculation anomalies, the portal establishes absolute transparency and equity—core pillars of **SDG 4 (Quality Education)**. --- ## 📂 System Architecture & Modular Structure Adhering strictly to the core principles of **PROG103 Structured Programming**, the system abandons monolithic scripting in favor of a clean **Separation of Concerns (SoC)** across five distinct modules: 1. **main.py**: The central application gateway orchestrating initial table connectivity verifications and launching the Tkinter loop. 2. **config.py**: The singular source of truth containing global variables, styling assets, fonts (Tahoma), and the institutional grade tier lookup scale. 3. **database.py**: The persistence infrastructure handling psycopg2 pipeline creation pools and pre-seeding the **20 mandatory validation rows**. 4. **logic.py**: The processing engine executing boundary mathematical checks, GPA lookups, and driving cross-platform PDF generation using fpdf2. 5. **gui.py**: The event-driven presentation layer managing user focus entry forms, tabular views, filter controls, and rendering **Matplotlib** charts. --- ##
